Представление (VIEW) - это виртуальная таблица, которая представляет собой сохраненный SQL-запрос. VIEW не хранит данные физически, а отображает результаты запроса в виде таблицы.

Представление можно сравнить с "сохраненным фильтром" или "окном" в данные:
Как окно в доме показывает только определенную часть улицы
Как фильтр в кофемашине пропускает только кофе, но не гущу
Как сохраненный поисковый запрос в интернет-магазине

Синтаксис создания view: 

CREATE [OR REPLACE] VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

WITH CHECK OPTION гарантирует, что все изменения через VIEW соответствуют условию WHERE в определении VIEW.



Зачем нужны view: 
Причина	            Описание	                            Пример
Упрощение	        Скрывают сложность JOIN 	            Вместо 10 JOIN каждый раз используем
Безопасность	    Скрывают конфиденциальные данные	    Показываем имя, но не зарплату
Согласованность	    Все используют единые правила расчета	Одинаковые формулы в отчетах
Абстракция	        Изменение структуры БД не ломает код	Меняем таблицы, VIEW остается
Производительность	Некоторые VIEW оптимизируются	        Часто используемые запросы в кеше


MATERIALIZED VIEW
Определение:
Материализованное представление - это физическая копия данных, которая хранится на диске и периодически обновляется.

Сравнение с обычным VIEW:

Характеристика	    Обычное VIEW	    MATERIALIZED VIEW
Хранение данных	    Не хранит	        Хранит физически
Скорость запросов	Зависит от таблиц	Быстро
Актуальность	    Всегда 100%	        Может быть устаревшей
Обновление	        Автоматически	    По расписанию или вручную
Использование	    Для простоты	    Для производительности


Когда использовать MATERIALIZED VIEW?
Тяжелые агрегатные запросы - подсчет сумм, средних значений
Отчеты за прошлые периоды - данные не меняются
Дашборды и аналитика - не нужна секундная точность
Звездные схемы (Data Warehouse) - предрасчетные таблицы фактов


Стратегии обновления 
Тип обновления	    Описание	                    Когда использовать
COMPLETE	        Полное перестроение	            Мало данных, редко меняется
FAST	            Только изменения	            Есть лог изменений, много данных
FORCE	            FAST если можно, иначе COMPLETE	Оптимальный вариант
ON COMMIT	        При каждом COMMIT	            Нужна 100% актуальность
ON DEMAND	        Вручную или по расписанию	    Для отчетов и аналитики


Оконные фукнции


Оконные функции — это функции, которые выполняют вычисления над набором строк, связанных с текущей строкой, без группировки результата в одну строку.

Главное отличие от GROUP BY:
GROUP BY — схлопывает строки в одну на группу
Оконные функции — сохраняют все строки, добавляя вычисленное значение

Синтаксис

ФУНКЦИЯ(колонка) OVER (
    PARTITION BY колонка_для_разбиения
    ORDER BY колонка_для_сортировки
    ROWS/RANGE BETWEEN ... AND ...
)

ВИДЫ ОКОННЫХ ФУНКЦИЙ
Агрегатные:
SUM(), AVG(), COUNT(), MIN(), MAX()

Ранжирующие:
ROW_NUMBER() — уникальный номер строки (1,2,3...)
RANK() — номер с пропусками после одинаковых значений
DENSE_RANK() — номер без пропусков
NTILE(n) — разбивает на n групп

Функции смещения:
LAG(колонка, n) — значение из строки n позиций назад
LEAD(колонка, n) — значение из строки n позиций вперед
FIRST_VALUE(колонка) — первое значение в окне
LAST_VALUE(колонка) — последнее значение в окне

Синтаксис рамок
ROWS BETWEEN [start] AND [end]
-- или
RANGE BETWEEN [start] AND [end]


UNBOUNDED PRECEDING — от начала
n PRECEDING — n строк назад
CURRENT ROW — текущая строка
n FOLLOWING — n строк вперед
UNBOUNDED FOLLOWING — до конца


-- Нумерация строк
ROW_NUMBER() OVER(ORDER BY колонка)
-- Ранжирование с пропусками
RANK() OVER(ORDER BY колонка)
-- Ранжирование без пропусков
DENSE_RANK() OVER(ORDER BY колонка)
-- Разбиение на группы
NTILE(n) OVER(ORDER BY колонка)
-- Предыдущее значение
LAG(колонка, n) OVER(ORDER BY колонка)
-- Следующее значение
LEAD(колонка, n) OVER(ORDER BY колонка)
-- Первое значение в окне
FIRST_VALUE(колонка) OVER(PARTITION BY ... ORDER BY ...)
-- Сумма по группе (сохраняя строки)
SUM(колонка) OVER(PARTITION BY группа)
-- Скользящая сумма
SUM(колонка) OVER(ORDER BY дата ROWS BETWEEN 1 PRECEDING AND CURRENT ROW)
