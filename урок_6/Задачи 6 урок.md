3. ЗАДАЧИ НА DELETE (УДАЛЕНИЕ ДАННЫХ)
Задача 3.1: Удаление отмененных бронирований
Удалите все бронирования со статусом 'cancelled', которые были созданы больше месяца назад.

Задача 3.2: Удаление неактивных официантов
Удалите официантов, у которых is_active = 0 и нет ни одного заказа.

Задача 3.3: Удаление посетителей без заказов
Удалите посетителей, которые зарегистрировались больше года назад и не сделали ни одного заказа.

Задача 3.4: Удаление непопулярных блюд
Удалите блюда, которые ни разу не заказывали (нет в order_items).

Задача 3.5: Очистка старых заказов
Удалите все заказы со статусом 'cancelled' старше 30 дней и связанные с ними позиции (учитывая каскадное удаление).

4. ЗАДАЧИ НА СОЗДАНИЕ ТРИГГЕРОВ
Задача 4.1: Триггер для автоматической даты регистрации
Создайте триггер, который автоматически проставляет текущую дату и время в поле registered_at при вставке нового посетителя, даже если это поле не указано в INSERT.

Задача 4.2: Триггер для проверки цены
Создайте триггер, который запрещает устанавливать цену блюда ниже 100 рублей или выше 5000 рублей.

Задача 4.3: Триггер для логирования удаления блюд
Создайте триггер, который при удалении блюда записывает информацию об этом в таблицу deleted_dishes_log (id, name, category_id, price, deleted_at, deleted_by).

Задача 4.4: Триггер для автоматического обновления суммы заказа
Создайте триггер, который при добавлении новой позиции в order_items автоматически пересчитывает общую сумму заказа в таблице orders.

Задача 4.5: Триггер для проверки бронирований
Создайте триггер, который запрещает бронировать столик на прошедшую дату (reservation_time должна быть больше или равна текущей дате).

Задача 4.6: Триггер для аудита изменений цен
Создайте триггер, который при изменении цены блюда записывает в таблицу price_changes_log старую и новую цену, дату изменения и имя пользователя.

Задача 4.7: Триггер для автоматического списания ингредиентов
Создайте триггер, который при добавлении позиции в заказ автоматически уменьшает количество соответствующих ингредиентов на складе (используя таблицу dish_ingredients).

Задача 4.8: Триггер для проверки уникальности телефона
Создайте триггер, который проверяет, что телефон нового посетителя уникален (хотя это можно сделать через UNIQUE constraint, но реализуйте через триггер).

Задача 4.9: Триггер для запрета удаления популярных блюд
Создайте триггер, который запрещает удаление блюд, которые были заказаны больше 5 раз.

Задача 4.10: Триггер для обновления времени изменения
Создайте триггер, который при обновлении любого поля в таблице dishes автоматически обновляет поле last_updated (предварительно добавьте это поле в таблицу).
