1. UNION — ОБЪЕДИНЕНИЕ ЗАПРОСОВ
1.1 Определение
UNION — это оператор языка SQL, который объединяет результаты двух или более SELECT-запросов в один результирующий набор.

1.2 Виды UNION
Оператор	    Описание
UNION	        Объединяет результаты и удаляет дубликаты
UNION ALL	    Объединяет результаты без удаления дубликатов

1.3 Правила использования UNION
Количество колонок во всех SELECT должно быть одинаковым
Типы данных соответствующих колонок должны быть совместимы
Имена колонок в результате берутся из первого запроса
ORDER BY можно применять только один раз в конце всего запроса
Скобки можно использовать для управления порядком выполнения

1.4 Когда что использовать
Ситуация	                            Рекомендация
Нужны уникальные записи	                UNION
Нужны все записи, дубликаты допустимы	UNION ALL (быстрее)
Точно нет дубликатов	                UNION ALL (быстрее)
Не уверены в дубликатах	                UNION

1.5 Другие операторы множеств
INTERSECT — возвращает общие строки из обоих запросов
MINUS (в Oracle) / EXCEPT — возвращает строки из первого запроса, которых нет во втором

2. EXISTS — ПРОВЕРКА СУЩЕСТВОВАНИЯ
2.1 Определение
EXISTS — это логический оператор, который проверяет, возвращает ли подзапрос хотя бы одну строку.

2.2 Синтаксис
sql
WHERE EXISTS (подзапрос)
WHERE NOT EXISTS (подзапрос)

2.3 Принцип работы
Подзапрос выполняется для каждой строки внешнего запроса (коррелированный подзапрос)
EXISTS возвращает TRUE, если подзапрос возвращает хотя бы одну строку
EXISTS возвращает FALSE, если подзапрос не возвращает строк
Важно: EXISTS проверяет только наличие строк, сами значения не важны

2.4 Особенности EXISTS
В подзапросе обычно пишут SELECT 1 или SELECT * — это не влияет на производительность
EXISTS останавливается при первом же совпадении (не сканирует все данные)
EXISTS работает эффективнее с индексами
NOT EXISTS может работать медленно, если множество результатов большое

2.5 EXISTS vs IN
EXISTS	                                IN
Проверяет наличие строк	                Сравнивает со списком значений
Работает с любыми подзапросами	        Работает с фиксированным списком или подзапросом
Останавливается при первом совпадении	Проверяет все значения
Эффективнее с большими наборами	        Может быть неэффективным
Лучше для коррелированных подзапросов	Лучше для статических списков
Не чувствителен к NULL	                Чувствителен к NULL (особенности)

2.6 NOT EXISTS
Возвращает TRUE, если подзапрос не возвращает строк
Полезен для поиска записей, которых нет в связанной таблице
Альтернатива: LEFT JOIN + проверка NULL

3. PL/SQL — ПРОЦЕДУРНЫЙ ЯЗЫК ORACLE
3.1 Определение
PL/SQL (Procedural Language / Structured Query Language) — это процедурное расширение SQL, разработанное Oracle Corporation. Оно добавляет в SQL процедурные элементы: переменные, условия, циклы, обработку ошибок.

3.2 Архитектура PL/SQL
text
PL/SQL Блок
    ↓
PL/SQL Engine (внутри Oracle)
    ↓
SQL-операторы → выполняются SQL-движком
Процедурный код → выполняется PL/SQL-движком

3.3 Структура PL/SQL-блока
PL/SQL-блок состоит из трех секций:

Секция	Обязательность	Назначение
DECLARE	Опционально	Объявление переменных, констант, курсоров, типов
BEGIN	Обязательно	Исполняемый код (основная логика)
EXCEPTION	Опционально	Обработка ошибок и исключений
END;	Обязательно	Завершение блока

3.4 Типы PL/SQL-блоков
Тип блока	            Описание	                                    Где хранится
Анонимный	            Не имеет имени, выполняется один раз	        Не сохраняется в БД
Процедура	            Именованный блок, может принимать параметры	    В БД
Функция	                Именованный блок, возвращает значение	        В БД
Пакет	                Группа связанных процедур/функций/переменных	В БД
Триггер	                Выполняется автоматически при событии	        В БД

3.5 Переменные в PL/SQL
Объявление переменных:
Переменные объявляются в секции DECLARE
Каждая переменная имеет имя и тип данных
Можно задать начальное значение (DEFAULT или :=)
Типы данных:

Скалярные: NUMBER, VARCHAR2, DATE, BOOLEAN, CHAR
Составные: RECORD, TABLE, VARRAY
Ссылочные: REF CURSOR
LOB: BLOB, CLOB, BFILE
Область видимости: Переменные видны только внутри своего блока
Вложенные блоки видят переменные внешних блоков

3.6 Константы
Значение нельзя изменить после объявления
Объявляются с ключевым словом CONSTANT
Должны быть инициализированы сразу

3.7 Присваивание значений
Оператор присваивания: := (двоеточие с равно)
SELECT INTO — для получения значения из БД

3.8 Управляющие конструкции
Условные операторы:
IF-THEN-END IF
IF-THEN-ELSE-END IF
IF-THEN-ELSIF-ELSE-END IF
CASE (простой и поисковый)

Циклы:
LOOP ... EXIT WHEN ... END LOOP — бесконечный цикл с выходом
WHILE condition LOOP ... END LOOP — цикл с предусловием
FOR counter IN [REVERSE] lower..upper LOOP ... END LOOP — цикл со счетчиком

3.9 Курсоры
Определение: Курсор — это указатель на область памяти, где хранится результат SQL-запроса. Позволяет обрабатывать строки по одной.
Типы курсоров:
Явные — объявляются программистом
Неявные — создаются автоматически для DML-операций
Операции с явным курсором:

DECLARE — объявление
OPEN — открытие
FETCH — выборка строки
CLOSE — закрытие
Атрибуты курсоров:

%FOUND — TRUE если последний FETCH вернул строку
%NOTFOUND — TRUE если последний FETCH не вернул строку
%ISOPEN — TRUE если курсор открыт
%ROWCOUNT — количество обработанных строк

Курсорные циклы:

Простой LOOP с FETCH
CURSOR FOR LOOP (автоматическое открытие, выборка, закрытие)

3.10 Исключения (Exceptions)
Определение: Исключения — это механизм обработки ошибок в PL/SQL.

Типы исключений:
Системные (предопределенные) — например, NO_DATA_FOUND, TOO_MANY_ROWS, ZERO_DIVIDE
Пользовательские — объявленные программистом
Исключения, связанные с ошибками — по коду ошибки

Обработка исключений:
Секция EXCEPTION содержит WHEN ... THEN ...
Можно использовать OTHERS для всех остальных исключений
Функции SQLCODE и SQLERRM для получения информации об ошибке
RAISE для генерации исключения

3.11 Транзакции в PL/SQL
COMMIT — сохранение изменений
ROLLBACK — отмена изменений
SAVEPOINT — точка сохранения
Автономные транзакции (PRAGMA AUTONOMOUS_TRANSACTION)

3.12 Динамический SQL
EXECUTE IMMEDIATE — для выполнения SQL, сформированного на лету
DBMS_SQL — более мощный пакет для динамического SQL

3.13 Встроенные пакеты Oracle
DBMS_OUTPUT — вывод информации (PUT_LINE)
DBMS_SQL — динамический SQL
DBMS_LOB — работа с большими объектами
UTL_FILE — работа с файлами
DBMS_SCHEDULER — планирование заданий

4. СВЯЗЬ МЕЖДУ ТЕМАМИ
UNION и EXISTS в PL/SQL
UNION можно использовать в курсорах для объединения данных из разных источников
EXISTS часто используется в PL/SQL для проверки условий перед выполнением операций
В процедурах и функциях можно динамически формировать UNION-запросы

6. ВАЖНЫЕ ОСОБЕННОСТИ ДЛЯ ЗАПОМИНАНИЯ
UNION:
UNION удаляет дубликаты, UNION ALL — нет
Количество и типы колонок должны совпадать
UNION ALL работает быстрее UNION

EXISTS:
Проверяет только наличие строк, значения не важны
EXISTS останавливается при первом совпадении
NOT EXISTS ищет отсутствие связей
EXISTS часто эффективнее IN для больших наборов

PL/SQL:
PL/SQL — это не SQL, а процедурный язык вокруг SQL
Блоки PL/SQL компилируются и хранятся в БД
Переменные объявляются в DECLARE, инициализируются в BEGIN
Исключения обрабатываются в секции EXCEPTION
Курсоры позволяют обрабатывать строки по одной
Процедуры выполняют действия, функции возвращают значения
